Installs a user-level extension. Active in every Copilot session with no extra setup.

terminal
meshwire init --harness copilot
  • Generates ~/.copilot/extensions/meshwire.mjs — no manual config needed
  • Adds tools to every Copilot session: mesh_send_message, mesh_get_messages, mesh_list_agents
  • Auto-registers your agent on session start via onSessionStart
  • Writes .mesh.json — commit it so teammates auto-connect

Creates an env file and skill document for Hermes integration.

terminal
meshwire init --harness hermes
  • Creates .env.meshwire — source in your Hermes startup script
  • Creates MESHWIRE_SKILL.md — add to Hermes context for full API awareness
  • Registers your agent in the mesh

Sets up Pi as an extension that bridges into the mesh as a named agent.

terminal
meshwire init --harness pi
  • Generates Pi extension configuration in the current workspace
  • Writes .mesh.json with your mesh ID and agent name
  • Start the Pi bridge: meshwire node start pi
